Drupal AMP Module

Drupal AMP Theme

AMP PHP Library

  • For our Drupal powered CMSs, we have this available tools to help build articles in AMP format
  • Supports Drupal 7 and Drupal 8
  • Can be enabled for any node type
  • Provides support for various AMP Components
    • [amp-ad],[amp-pixel],[amp-img],[amp-video],
      [amp-analytics] and [amp-iframe]
  • <body> field presents a special problem, since it is likely to contain lots of invalid markup
  • Includes a ExAMPle Subtheme, which shouws how to set custom styles with AMP
  • A custom subtheme can then be made that extends the AMP-Base theme to create custom design for site's AMP pages
  • Open source and pure PHP library
  • Built and maintained by Lullabot
  • Converts Regular HTML to AMP HTML
    • <img> tag is converted to an <amp-img> tag
    • <iframe> tag is converted to an <amp-iframe> tag
    • <audio> tag is converted to an <amp-audio> tag
    • <video> tag is converted to an <amp-video> tag
    • Twitter embed code for tweets is converted to an <amp-twitter> tag.
    • Instagram embed code for instagrams is converted to an <amp-instagram> tag.
    • Youtube embed code for videos is converted to an <amp-youtube> tag
  • Removes illegal tags and attributes within <body>

Conclusion

  • AMP provides the discipline for creating a faster, cleaner mobile experience that can better attract and hold onto mobile readers
  • Short-term Goals
    • BH, BT, Stomp with their mostly basic article layouts are best candidates for AMP
    • Implement better SEO and AMP analytics
  • Long-term Goals
    • The website is no longer the digital end product, hence:
      • Start building articles not just for websites (agnostic content)
      • Implement stricter adherence to semantics
      • Detailed tagging
    • Preparation for Google Subscription Tool and similar formats
    • AMP also works for E-commerce and online directories
  • Keyword here is discipline, we are not obligated to use AMP but the performance benefits that AMP structure is providing should guide us on how we should present our stories and build our sites
  • Many of our sites have a somewhat basic article layout so it should be fairly easy to convert them to AMP format
  • A more detailed study of our mobile traffic and search referals should help us optimize how much effort we should devote to providing AMP formats
  • of course SEO is key here (at least for now) because unless our content appear in search results, readers will not be able to find our content
  • Native Payment API on the browser for better conversion
